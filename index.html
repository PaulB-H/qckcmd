git add .
Stage all saved files for next commit
git add FILENAME
Stage specific file(s) for next commit
Specify dir as DIR/DIR/FILENAME 
git add -p
 -- -- -- --
(1/10) Stage this hunk [y,n,q,a,d,j,J,g,/,s,e,?]? ?
y - stage this hunk
n - do not stage this hunk
q - quit; do not stage this hunk or any of the remaining ones
a - stage this hunk and all later hunks in the file
d - do not stage this hunk or any of the later hunks in the file
g - select a hunk to go to
/ - search for a hunk matching the given regex
j - leave this hunk undecided, see next undecided hunk
J - leave this hunk undecided, see next hunk
s - split the current hunk into smaller hunks
e - manually edit the current hunk
? - print help
Add to staging specific hunks from within a file(s)
Note ability to split hunks into smaller parts 
git commit -m "SUBJECT" -m "DETAILS"
SUBJECT recommend character length = 50, max = 72. DETAILS put anything more. 
git push origin TARGET
Pushes to specified remote, typically "main"
git reset
Remove from staging anything that has not been committed. 
git reset HEAD~1
Locally remove last commit, un-stage all changes. 
git reset --soft HEAD~1
Locally remove last commit, changes remain staged. 
git reset --hard HEAD~1
Locally remove last commit, DELETE any changes.

NGINX

nginx
High Performance Load Balancer, Web Server, & Reverse Proxy

Server Block Example

Sections managed by certbot are inserted upon SSL generation

The first section listens on port 443 for HTTPS and serves index/root dir


The second section listens on port 80 for HTTP requests, and redirects them to HTTPS 

server {
  # Used to select server block for processing
  server_name mydomain.ca www.mydomain.ca;

  # Configures publicly served root dir
  root /var/www/mydomain;
	
  # Configures index file to be served
  index index.html index.htm;
	
  # Below managed by Certbot
  listen 443 ssl;
  ssl_certificate /etc/letsencrypt/live/www.mydomain.com/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/www.mydomain.com/privkey.pem;
  include /etc/letsencrypt/options-ssl-nginx.conf;
  ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;
} 
	
server {
  # Redirect if not https
  if ($host = www.mydomain.ca) {
    return 301 https://$host$request_uri;
  } # Managed by Certbot
  if ($host = mydomain.ca) {
    return 301 https://$host$request_uri;
  } # Managed by Certbot
	
  listen 80;
  server_name mydomain.ca www.mydomain.ca;
	
  return 404; # managed by Certbot
	
}

ln -s /etc/nginx/sites-available/mydomain.ca /etc/nginx/sites-enabled/
Create symbolic link from specified file into dir at:
/etc/nginx/sites-enabled/

Run this once base server block has been setup. After the command is run, the created file will always match the contents of the reference file, no need to run the command again. 
sudo systemctl restart nginx
